The recommended browser onboarding flow derives the recovery phrase from a wallet signature (EIP-712). Sign the same payload twice and compare — if both signatures match, derive the mnemonic via HKDF. If the wallet produces non-deterministic signatures, fall back to manual mnemonic entry. Either way, require the recovery phrase to be saved before the first deposit.
